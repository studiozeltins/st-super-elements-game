  <action>Human-verified two-client playtest — see how-to-verify for the exact steps. No code changes; this gates the phase before /gsd-verify-work.</action>
  <what-built>
    The full player-party feature: online-players list → slide-out player sheet with
    "Aicināt manā pulkā" / "Lūgt pievienoties pulkam" / "Pamest pulku"; invite/request toast
    (10s, Pieņemt/Noraidīt); missed-invites under Settings; live party roster (character + role
    badge + leader crown + online dot); server enforcement of one-party-per-player, the 4-cap,
    oldest-joined promotion, disband-on-last-leave, and disconnect-persists-membership.
  </what-built>
  <how-to-verify>
    Two browser clients (A and B) against the local build (laragon-served dist over LAN, or two
    windows). Verify each behavior:
    1. FORM: A opens B in the player sheet → "Aicināt manā pulkā". B sees a .party-toast → Pieņemt.
       Both A and B now show a 2-member roster with each member's active character + role badge; A
       has the leader crown ♛; online dots are filled for both.
    2. ASK-TO-JOIN: A third client C uses "Lūgt pievienoties pulkam" on A → A (leader/recipient)
       gets the request toast → Pieņemt → roster shows 3/4.
    3. MISSED INVITES: send an invite, let the toast expire (~10s, do not act) → the invite still
       appears under Settings → NOKAVĒTIE AICINĀJUMI and can be accepted there.
    4. CAP + DOUBLE-JOIN: fill to 4/4 → the sheet's invite action is disabled with
       "Pulks ir pilns (4/4)"; attempting a 5th accept is rejected; a player already in the party
       cannot double-join.
    5. PROMOTE: leader A leaves ("Pamest pulku" → confirm) with ≥2 members remaining → the crown
       moves to the OLDEST-joined remaining member.
    6. DISBAND: all members leave → the party + its pending invites are gone (empty-state roster).
    7. DISCONNECT PERSISTS (D-04): a member closes their tab → the roster shows them OFFLINE
       (hollow ○ dot) but STILL a member; on reconnect they are back online in the same party.
    Report PASS/FAIL per numbered item.
  </how-to-verify>
  <resume-signal>Type "approved" if all 7 pass, or describe which item(s) failed.</resume-signal>
</task>
