  <action>
    Create createTelegraphSystem(scene: THREE.Scene, getGroundHeight: (x: number, z: number) => number)
    following the createEffectSystem factory convention (~150 LOC budget). Per D4-14:

    - One telegraph per unit_attack row with state === windup (1) or strike (2), keyed
      `${unitKind}:${unitId}` in a Map, reconciled by a syncAttacks(rows, isUnitAlive) method using
      the createEntityRenderer seen-set pattern. isUnitAlive(unitKind, unitId) is a caller-supplied
      predicate; rows whose unit is dead/missing get their meshes removed immediately (no telegraph
      outlives its goliath).
    - OUTLINE: THREE.RingGeometry(radius - 0.25, radius, 48), MeshBasicMaterial color 0x86e2ff,
      transparent, opacity ~0.85, depthWrite: false, side DoubleSide, rotation.x = -PI/2, position
      y = getGroundHeight(landingX, landingZ) + 0.06. Rim width 0.25 world-units (>= the 0.2 ANIM-02
      floor for 440px legibility). Appears instantly at windup entry, full radius.
    - FILL DISC: THREE.CircleGeometry(radius, 48), same material family at opacity ~0.3, scaled by
      fillFraction via mesh.scale.setScalar — the fill percentage IS the countdown.
    - TIMING (ANIM-01): when a row first arrives (or transitions into windup), capture
      arrivalPerfMs = performance.now() and the row's startedAtMicros/strikeAtMicros. Each update()
      frame: elapsedMicros = BigInt(Math.round((performance.now() - arrivalPerfMs) * 1000)); fill =
      clamp(Number(elapsedMicros) / Number(strikeAtMicros - startedAtMicros), 0, 1). This derives
      timing from the server row's duration with an arrival-anchored offset — the fill completes
      exactly one windup-duration after the row arrived (RTT-late = defender-friendly). NEVER start
      an independent countdown constant on the client; the only duration used is
      strikeAtMicros - startedAtMicros from the row. Use THREE.MathUtils.clamp (the only sanctioned
      interpolation helper).
    - RIM FLASH at strike: when a row's state transitions windup -> strike (detected in
      syncAttacks) or when flashStrike(unitKey) is called externally, pop the outline opacity to
      1.0 and scale to ~1.08 then decay back over ~0.2s in update(). Remove meshes entirely when
      state leaves strike (recovery/idle) after the flash decays.
    - dispose() removes all meshes from the scene and disposes geometries/materials via the
      existing engine disposeObject helper.
    Keep it geometry-only (no shaders) — CONTEXT discretion resolved: geometry is cheaper and the
    pixel filter is the constraint, not fill smoothness.
  </action>
