<objective>
Surface the combat role as a single read-only text badge on the two character identity surfaces, reusing the existing bespoke HUD tag design verbatim (per 04-UI-SPEC, APPROVED — no new design language). The badge is a passive `<span>` (not a button; no hover/press/focus), color-blind-safe (Latvian text label + redundant color), and defensively renders nothing for an invalid/missing role.

Depends on Plan 04-01 for the `role` field on every character and the exported `ROLE_META` map.

Purpose: Make the role visible to players on the character screen and character card.
Output: four `--role-*` CSS tokens, two badge CSS classes, and a badge rendered on CharacterScreen + CharacterSheet.
</objective>
