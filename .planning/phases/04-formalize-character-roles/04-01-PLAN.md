<objective>
Add the intrinsic `role` combat-role attribute (tank | dps | healer | support) to every character in both the client catalog (`characters.ts`) and the server const mirror (`CHARACTER_STATS` in `spacetimedb/src/index.ts`), guarantee the two stay in lockstep via `serverSync.test.ts` (INV-5), and deploy the additive change to the local module so Phase 6 raid logic can read `statsFor(id).role` at runtime.

Per D REQ-combat-roles: role is stored server-side (const mirror, NOT a table — `CHARACTER_STATS` is a plain object literal, so there is no migration, no column, no seed reducer, and no binding regeneration). The exact 17-character mapping is prescribed by the requirement and lifted verbatim from 04-RESEARCH.md.

Purpose: Establish the server-visible role data + parity guarantee that Phase 5 (party roster) and Phase 6 (raid enforcement) build on.
Output: role on 34 entries (17 client + 17 server) + fallback, two new parity tests green, additive local publish.
</objective>
