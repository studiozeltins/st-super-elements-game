# Phase 5 вЂ” UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope of this phase's UI deliverable:** the client surfaces for **player parties** (invite-based,
Fortnite-style) defined by CONTEXT.md D-01..D-10. Five surfaces:

1. **Player-target slide-out sheet** (`.player-sheet`) вЂ” opened by tapping/clicking a player; carries the
   symmetric "AicinДЃt manДЃ pulkДЃ" / "LЕ«gt pievienoties pulkam" actions, plus "Pamest pulku" when partied.
2. **Invite / join-request toast** (`.party-toast`) вЂ” slides in from the right edge, Accept / Decline,
   ~10s client-side auto-dismiss, non-intrusive.
3. **Missed-invites list** (`.missed-invites`) вЂ” a calm surface mounted under Settings to act on
   dismissed/missed invites later.
4. **Party roster panel** (`.party-roster`) вЂ” one row per member: active character + role badge +
   leader crown + online dot.
5. **Party entry point in the HUD** (`.hud__party-chip`) вЂ” opens the roster panel.

Everything server-side (`party` / `party_member` / `party_invite` tables + reducers) is non-visual and
out of this contract's scope. This phase **reuses the existing bespoke HUD design system verbatim** and
introduces **no new design language or palette** (per the phase brief and CONTEXT "match existing patterns").

**Naming caveat (mirrors the code-level collision):** the existing gacha "Komanda" tab / `PARTY_SIZE`
is the personal **4-character team**. The new **player group** must NOT reuse "Komanda". User-facing Latvian
noun for the player party is **"Pulks"** (band/squad) throughout вЂ” distinct from the character team, and
consistent with the server's `RAID_PARTY_SIZE` separation.

**Component metrics (reuse the shipped values verbatim вЂ” do NOT re-derive):**

- Button / row / chip radius: **8px** (matches `.btn`, `.toggle`, `.icon-btn`).

`6px`/`10px`/`14px` gaps appear inside reused blocks (`.modal__*`, `.btn` gap 6px, `.sheet__meta` gap 14px).
When a new element sits beside a shipped one, match the neighbour's value rather than the canonical token.

## Component Contracts

### 1. Player-target slide-out sheet вЂ” `.player-sheet`

- **Trigger (flagged in CONTEXT for research):** primary = tap the on-screen 3D player avatar (raycast);
  fallback = an online-players list entry. **Both triggers open the same `.player-sheet`** вЂ” the sheet is
  trigger-agnostic. If raycast hit-testing proves impractical, the online-players list is the acceptable
  surface (CONTEXT "Claude's Discretion"). Spec the sheet identically regardless.
- **Structure:** scrim `.player-sheet__scrim` (`position:absolute; inset:0; z-index:14; background:rgba(4,7,5,0.82)`,
  click-to-close, `cursor:pointer`) + `.player-sheet__panel` docked to the **right edge**
  (`position:absolute; top:0; right:0; height:100%; width:min(360px, 92vw); border-radius:12px 0 0 12px;
  background:var(--panel); border-left:1px solid var(--line)`). Rounded per D-09.
- **Contents:** header (player name 18px/700, active character name + element color, role badge via
  `ROLE_META`, online dot); action stack (`gap:16px`): primary invite в†’ secondary ask-to-join в†’ (danger
  Leave when partied). Close affordance: `.icon-btn` `вњ•` top-right, ESC, and scrim click.
- **Actions visibility:** if the viewer and target share a party в†’ show only `Pamest pulku`. Otherwise show
  invite + ask-to-join; disable invite (with helper copy) when the viewer's party is full.
- **Touch/focus:** each action в‰Ґ44px tall; first action auto-focused on open; focus trapped within the panel
  (use Radix Dialog wrapper OR replicate its trap вЂ” prefer wrapping in `Modal` semantics).

### 2. Invite / join-request toast вЂ” `.party-toast`

- **Placement:** slides in from the **right edge** (opposite side from where the sheet's trigger/roster live),
  stacked top-right under the wallet: `position:absolute; top:88px; right:max(16px,env(safe-area-inset-right));
  z-index:30`. Multiple toasts stack downward with `8px` gap.
- **Body:** `background:var(--panel); border:1px solid var(--line); border-radius:12px; padding:12px`.
  Message (14px/400) + action row (`gap:8px`): `PieЕ†emt` (primary) / `NoraidД«t` (ghost), each в‰Ґ44px.
- **Auto-dismiss:** ~10s client-side timer (D-07); on expiry the toast slides out and the item drops into the
  missed-invites list. Timer is **client-only** вЂ” the invite is NOT server-expired (D-08). Hovering/focusing
  the toast pauses the auto-dismiss (a11y: don't time out a keyboard user mid-read).
- **Non-intrusive:** never blocks input, never covers the joystick/action cluster, `pointer-events` only on
  the buttons вЂ” the rest of the toast is passive.

### 3. Missed-invites list вЂ” `.missed-invites`

- **Mount:** inside `SettingsScreen` as a new section (`.settings__section` heading `NOKAVД’TIE AICINДЂJUMI`),
  a calm/non-intrusive location (D-07). Left to planning whether a dedicated small panel is added later;
  Settings is the contract default.
- **Row:** one `.missed-invites__item` per pending invite/request the viewer can still act on: message (14px)
  + `PieЕ†emt` / `NoraidД«t` (reuse `Button`). Empty state copy above.
- Rows are driven by the same `party_invite` rows (client-derives "missed" = not currently shown as a live
  toast); acting from here calls the same accept/decline reducers.

### 4. Party roster panel вЂ” `.party-roster`

- **Surface:** a compact panel opened from the HUD entry chip. Use `Modal` (`size="sm"`) OR an in-HUD
  slide-down panel reusing `.panel`/`.modal__panel` styling. Title `Pulks В· {n}/4`.
- **Row (`.party-roster__row`, one per member):** online dot (`в—Џ accent` / `в—‹ muted`) В· member name (14px) В·
  active-character name (element color) В· **role badge** (reuse `ROLE_META` + `.sheet__role` box) В·
  leader crown `в™›` (`var(--gold)`, `aria-label="Pulka vadonis"`) on the leader row only. Row padding `11px 12px`,
  radius `8px`, `background:var(--panel)`.
- **States:** empty (no party) в†’ empty-state copy + hint; offline member в†’ row `opacity:0.5` + hollow dot;
  full в†’ title reads `4/4`. Leader always rendered first, then members by `joinedAt`.
- Role/character/online all read from existing data (`player.online`, `player.activeCharacterId`,
  `statsFor(activeCharacterId).role`) вЂ” no new server field.

### 5. HUD entry chip вЂ” `.hud__party-chip`

- **Placement:** in `.hud__meta-row` beside the zone badge (`.hud__top-left`), reusing `.hud__zone` box
  language (panel bg, `1px` line border, 10px/700 uppercase). In party в†’ `PULKS {n}/4`; no party в†’ `PULKS`.
- **Interaction:** opens the roster panel (surface 4). Hover/focus border в†’ `var(--accent)`

- Do NOT collide with the existing `вљ‘` "Komanda" (character-team) quick-button вЂ” this is the **player** party,
  a separate text chip on the left cluster.

## Animation & Motion Contract

Motion is **transform/opacity only** (matches the shipped HUD rule) вЂ” never animate layout/color for entry.

| Element | Enter | Exit / auto | Easing / duration |
|---------|-------|-------------|-------------------|
| `.player-sheet__panel` | `translateX(100%) в†’ 0` + scrim `opacity 0в†’1` | reverse on close | `cubic-bezier(0.2,0.9,0.2,1)`, 0.22s (matches `.modal__panel`) |
| `.party-toast` | `translateX(120%) в†’ 0`, `opacity 0в†’1` | at ~10s: `translateX(0 в†’ 120%)`, `opacity 1в†’0`, 0.3s | ease-out, 0.3s in |
| `.party-roster` (as Modal) | inherit `.modal__panel` `modal-rise` | inherit | as shipped |
| Roster row promote/leave | `opacity` cross-fade only (no reflow jump) | вЂ” | 0.15s ease |

**Reduced motion (mandatory вЂ” add these blocks):**
```css
@media (prefers-reduced-motion: reduce) {
  .player-sheet__panel,
  .party-toast { animation: none; transition: opacity 0.12s linear; transform: none; }
}
```
Under reduced motion the sheet and toast **fade only** (no translate); auto-dismiss timing is unchanged, and
the manual close/accept/decline controls remain the primary path.

## Accessibility Contract

- **Touch targets:** all NEW interactive controls (sheet actions, toast Accept/Decline, roster open, entry
  chip on coarse pointers) в‰Ґ **44Г—44px**. Existing 34вЂ“36px controls (`.icon-btn`, `.hud__quick-btn`) are
  unchanged and out of scope.
- **Keyboard/focus:** sheet and roster trap focus and restore it to the trigger on close (Radix Dialog via
  `Modal`, or equivalent). ESC closes both. First actionable control auto-focused on open. Global
  `:focus-visible` ring (`2px var(--accent)`) applies вЂ” do not remove it.
- **Toast a11y:** toast container `role="status"` / `aria-live="polite"` so it's announced without stealing
  focus; hover/focus pauses the 10s auto-dismiss.
- **Non-color-alone:** online state = dot shape (`в—Џ`/`в—‹`) **and** `aria-label`, not color alone; leader =
  crown glyph **and** `aria-label`; role = badge label text **and** color.
- **Screen-reader labels:** every glyph-only control (`вњ•`, `в™›`, dot) carries an `aria-label`; decorative
  glyphs use `aria-hidden`.
