<objective>
Complete the damage-number system so the server-owned crit truth reaches the screen, and deploy + verify the whole phase on a live migrated DB.

- Own hits: instant, display-only local prediction (0 latency) for smooth feel; the server `enemy_hit` event upgrades ONLY the attacker's crit to the big red number and is otherwise suppressed for own hits (D2-01/D2-02).
- Other players' hits on shared enemies: event-driven numbers for full co-op visibility (D2-02).
- Remove the legacy server HP-delta number spawn (which would now double/triple-draw) while KEEPING the health-bar reveal (RESEARCH §Damage-Number Reconciliation).
- Deploy the additive schema to local + maincloud, regenerate + build, then a two-client migrated-DB playtest confirms per-character crit divergence (CRIT-02) and no visual regression (CRIT-04), and that `enemy_hit` rows + `skillReadyAt` state exist on a migrated (not fresh) DB.

Purpose: Delivers the visible crit spectacle (CRIT-04) on top of Plan 02's server authority, and closes the phase with the migrated-DB + two-client verification the milestone requires.
Output: Reworked client number rendering (local prediction + event-driven), the `enemy_hit` subscription/onInsert, a new `spawnWorldNumber` game handle, deployed schema on both environments, and a human-verified playtest.
</objective>
