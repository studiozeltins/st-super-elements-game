<objective>
The atomic server-authority slice: make the server own base damage AND the crit roll for enemy/goliath hits, delete the client crit roll, and swap the reducer signatures to intent — all in ONE slice so the build never half-migrates (RESEARCH Pitfall 4).

- `attackEnemies`/`attackRay` drop the `damage` arg, add `isSkill`; a new `resolvePlayerHit` glue helper computes base (via `computeBaseDamage` from `CHARACTER_COMBAT` + active constellation/transcend), rolls crit (`rollCrit(..., () => ctx.random())`), applies resistance, clamps to `MAX_HIT_DAMAGE`, and emits an `enemy_hit` event row per landed hit (CRIT-02, CRIT-05, CRIT-06).
- `castSkill` becomes the authoritative skill rate-limiter: reject while on cooldown, else set `skillReadyAtMicros` + a hit `skillWindowEndsAtMicros`; `isSkill` hits get the uncapped skill multiplier only inside that window, else downgrade to basic (D2-03, SC4).
- Combo is clamped to `MAX_COMBO_FOR_GEMS` before the uncapped skill ramp (D2-04).
- Client: delete `rollDamage`/`CRIT_CHANCE`/`CRIT_MULTIPLIER`, send intent, regenerate bindings, keep the build green.

Purpose: Closes the damage-spoof (CRIT-06) and crit-spoof (CRIT-02) holes that make the PVP shard-theft loop exploitable — this is the security core of the phase.
Output: Reworked reducers + `enemy_hit` table + `skillReadyAt`/window player state + gated `castSkill`, and a green client build that sends intent. NOTE: the crit NUMBER visual + local prediction land in Plan 03 — after this plan, enemy numbers still come from the existing server HP-delta path (always 'normal'), which is no worse than today (no regression).
</objective>
