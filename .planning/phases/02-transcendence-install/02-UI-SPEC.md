## Spacing Scale

The codebase does **not** use a strict 8-point scale (it mixes 10/11/12/13/15px type and 5/10/12px padding). This phase conforms to the existing `.con-item` treatment rather than imposing a new scale.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon‚Äìnumber inline gap |
| sm | 8px | Compact element spacing |
| md | 12px | `.con-item` internal gap + row-to-row gap (existing exemplar) |
| lg | 16px | Section padding |

**Phase-specific values (locked, matching exemplars):**
- Transcend control padding: **10px 12px** (identical to `.con-item`, index.css:3032).
- Gap between the T-badge and the cost/label inside the control: **12px** (matches `.con-item` gap, index.css:3030).
- `T{n}` status-badge padding: **5px 10px** (identical to `.cchar__active-tag`, index.css:2810).
- Helper/gate text top spacing: **12px** dashed rule (reuse `.ctab__soon` treatment, index.css:2856).

Exceptions: inherits existing non-4-multiple type sizes (10/13/15px) from the exemplars for consistency; do not "fix" them here.

## Color

Existing token palette (`src/index.css` `:root`, lines 17‚Äì25), mapped to the 60/30/10 split. **This phase introduces zero new color values.**

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--ink` `#0c110e` | Screen fill, canvas backdrop |
| Secondary (30%) | `--panel` `rgba(12,17,14,0.72)` + `--line` `rgba(232,245,224,0.16)` | Panels, cards, the control's default border/background |
| Accent ‚Äî chrome (green) | `--accent` `#7ec843` | UI chrome only: kickers, active tab/rail, focus rings, `AKTƒ™VS VARONIS` tag. **NOT used by this phase's control.** |
| Accent ‚Äî constellation (gold) | `--gold` `#f2c14e` | Gems wallet + constellation (`C{n}`) rows. **NOT used by this phase's control.** |
| Accent ‚Äî transcend (purple) | `--shard` `#9333ea` | **This phase's family color:** the shard chip (existing), the Transcend/Install button (enabled + at-cap), the `T{n}` badges, and the cost glyph `‚óà`. |
| Destructive | `--danger` `#ff5c3c` | Destructive actions only ‚Äî **none in this phase; do not use for the "can't afford" gate.** |

**Accent reserved-for (this phase):** `--shard` purple is reserved exclusively for transcendence-family surfaces ‚Äî the Transcend button (enabled fill + at-cap fill), the `T{n}` badge, and the `‚óà` cost glyph. This creates the semantic contract **purple = spend shards ‚Üí gain transcend power**, visually distinct from gold (constellation/gems) and green (UI chrome). Never color the constellation `C{n}` rows purple, and never color the transcend control gold.

**Disabled/gate coloring (locked, no `--danger`):** the "insufficient shards" and "below C6" gates are communicated by **dimming** (`opacity: 0.45`, mirroring `.con-item--off`, index.css:3050) plus `--muted #93a48e` helper text ‚Äî **not** by `--danger`. `--danger` stays reserved for genuinely destructive actions (none here).

## Interaction & States

The Install control has **one enabled state and three distinct disabled states**, each with its own affordance and copy so the player always knows *why* it is blocked.

| State | Condition | Affordance | Copy (Latvian) |
|-------|-----------|-----------|-----------------|
| **Enabled** | `constellation >= 6` AND `transcendLevel < MAX` AND `transcendShards >= cost` | Full-color `--shard` purple button, interactive, chevron/cost pill lit. | Title `PƒÄRSNIEG≈ÝANA`, CTA cost `{cost} ‚óà`, badge `T{n}` (or `C6` at level 0). |
| **Gate: below C6** | `constellation < 6` | Locked look: `opacity: 0.45` (mirror `.con-item--off`), `disabled`, no chevron, muted lock glyph. Badge shows `C{constellation}`. | Label `Vajadzƒ´gs C6`; helper (`--muted`): `Atbloƒ∑ƒì visas 6 zvaigznes, lai sƒÅktu pƒÅrsnieg≈°anu.` |
| **Gate: at cap** | `transcendLevel >= MAX_TRANSCEND_LEVEL` (10) | Celebratory "complete" look: solid purple filled badge (`--shard`), `disabled`, no cost, no chevron. Not error-toned. | Label `Maksimums ¬∑ T10`; helper (`--muted`): `Sasniegts pƒÅrsnieg≈°anas maksimums.` |
| **Gate: not enough shards** | `constellation >= 6` AND `transcendLevel < MAX` AND `transcendShards < cost` | CTA stays "primed" (cost `{cost} ‚óà` visible) but `disabled` + dimmed (`opacity: 0.45`). No `--danger`. | Cost `{cost} ‚óà` shown; helper (`--muted`): `Vajag vƒìl {cost ‚àí transcendShards} ‚óà.` (explicit shortfall) |
| **Success (post-tap)** | Reducer succeeds; subscribed rows update | Shard chip pulses (**reuse existing `.wallet-chip--pulse`**, index.css:1111 ‚Äî already shipped Phase 1); badge increments `T{n}‚ÜíT{n+1}`; cost re-renders to `nextLevel+1`. | No toast required; the counter drop + badge bump are the feedback. |
| **Reducer rejection (race)** | Client gate passed but server `SenderError` (e.g. stale subscription) | No crash; the subscribed rows self-correct on the next push. No error toast required. | none (self-healing); optional silent `console.warn`. |

**Confirmation:** single-tap, **no confirmation dialog** ‚Äî consistent with the existing one-tap constellation toggles (`.con-item`, line 348) and low early-level cost. Deliberate decision; a press-to-arm confirm is out of scope (may revisit if high-level mis-taps prove costly in playtest).

**Disabled distinctness requirement (checker will verify):** the three disabled states must be visually and textually distinguishable ‚Äî *below C6* = locked/goal-setting, *at cap* = filled/complete, *not enough shards* = primed-but-dimmed with an explicit numeric shortfall. Do not collapse them into one generic greyed button.
